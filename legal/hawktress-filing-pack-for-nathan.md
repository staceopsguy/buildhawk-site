### Screen 5 — Goods/services (Class 42)

- Class number: **42**
- Number of classes: **1** (single class only — do NOT add Class 9 or 35 at this stage)
- Use the **picklist search** for each of the terms below. Type the bolded keyword into the picklist search, tick the closest matching picklist entry, and add it to the specification. Do this for each row:

| # | Picklist search keyword | What you're picking |
|---|---|---|
| 1 | `software as a service` | The umbrella SaaS-hosting term |
| 2 | `cost estimation` | Cost estimation software service |
| 3 | `project management software` | Construction/project management SaaS |
| 4 | `workflow management` | Workflow management software |
| 5 | `data analysis` | Cost data analysis services |
| 6 | `online non-downloadable software` | Hosted application access term |
| 7 | `construction industry` | Industry tag (if available as picklist modifier) |

If the picklist offers an exact-match phrase like *"Software as a service [SaaS] services featuring software for use in [field]"* — pick that and fill in the field with **construction cost estimation, construction project management, and workflow management**.

> **Rule:** stay inside picklist terms. The moment you write your own free text, the fee jumps from $250 to $400 AND examiner objection risk goes up. If the picklist doesn't have a perfect phrase, pick the closest match — the spec doesn't have to be perfect, it has to be defensible.
